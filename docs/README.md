## Coding
### Commit Convention
* commit 메시지 언어 : 한글
* feat() : 기능 추가
* refactor() : 서비스에 영향이 가는 구조 개선
* fix() : 에러가 나는 부분 해결
* docs() : document 파일 관련
* test() : 테스트 코드만 추가하거나 바꿀 때
* style() : 들여쓰기, 변수명, 메서드명 수정 및 기타 수정 사항
* chore() : 정말 사소한 커밋
* commit 단위 : 기능 단위로 커밋을 하되, 테스트와 기능 커밋 구분

### 코딩 컨벤션
* 모든 변수 앞에는 final을 붙인다.
* getter setter를 지양하기
* validation 검증 시점은 추후에 이야기 하도록...
* 드라이버 네비게이션은 일단 기능 주기로 바꾸기
* 일단은 돌아가는 코드를 만들자
